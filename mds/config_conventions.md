# コード規約 (Config Conventions)

## Rust コーディング規約

### 基本原則
- Rust公式のコーディング規約（rustfmt）に従う
- `cargo fmt` でフォーマットを統一
- `cargo clippy` でリントチェックを実施

### 命名規則
- 関数・変数: `snake_case`
- 型・構造体・列挙型: `PascalCase`
- 定数: `SCREAMING_SNAKE_CASE`
- モジュール: `snake_case`

### エラーハンドリング
- `Result<T, E>` を適切に使用
- `?` 演算子を活用してエラー伝播
- カスタムエラー型を定義して使用

### テスト
- すべての公開関数にテストを記述
- テストは `/tests/` ディレクトリに配置
- ユニットテストと統合テストを分離
- TDD原則に従い、テストファーストで実装

### ドキュメント
- 公開APIには `///` でドキュメントコメントを記述
- 複雑なロジックにはインラインコメントを追加

### パフォーマンス
- 不要なアロケーションを避ける
- 可能な限り参照を使用
- 非同期処理は `tokio` または `async-std` を使用

### 依存関係
- 最小限の依存関係に留める
- セキュリティとパフォーマンスを重視

## Python コーディング規約

### 基本原則
- PEP 8 に従う
- `black` でフォーマット（使用する場合）
- 型ヒントを使用

### 命名規則
- 関数・変数: `snake_case`
- クラス: `PascalCase`
- 定数: `SCREAMING_SNAKE_CASE`

### エラーハンドリング
- 適切な例外処理を実装
- エラーメッセージを明確に

## プロジェクト構造

```
web2ws/
├── src/           # Rustソースコード
├── tests/         # TDDテストコード
├── python/        # Python受信クライアント
├── mds/           # ドキュメント
└── Cargo.toml     # Rust依存関係
```
